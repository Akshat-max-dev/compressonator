Steps used to rebuild Qt and QScintalla on Windows.  This assumes that the full 4.7.4 tree is checked in to PerForce.  If this changes, then the first step will be to get a full build tree of Qt 4.7.4 from somehwere (Qt website?)

QT

1)	full sync qt 4.7.4
2)	subst q: c:\amdtools\main
3)	Open VS 2010 command prompt
4)	In Q:\common\lib\ext\qt\4.7.4:
	a)	attrib -r /s *.* (or checkout using P4)
	b)	del /s *.pdb (got an error if existing .pdbs were there)
	c)	configure.exe -qt-sql-sqlite -no-qt3support -no-dbus -no-phonon -no-phonon-backend -webkit -debug-and-release -opensource -shared -qtlibinfix AmdDt474
	d)	'y'
	e)	nmake
5)	In Q:\common\lib\ext\qt\4.7.4\lib:
	a)	move *.lib *.dll *.pdb *.exp win32
	b)	del *.ilk

QScintilla (ignore the GPL version)

1)	copy Q:\common\lib\ext\qt\4.7.4\lib\win32\*.lib Q:\common\lib\ext\qt\4.7.4\lib\ (to copy QT libs back to where QScintilla needs them)
2)	Open VS 2010 command prompt
3)	From the Common\lib\ext\QScintilla\2.5.1\Qt4 directory:
4)      Edit qscintilla.pro and change the "Release:TARGET = qscintillaAmdDt4742" and "Debug:TARGET = qscintillaAmdDt474d2"
        so that the "AmdDt474" string matches the string passed to the -qtlibinfix switch above
5)	..\..\..\qt\4.7.4\bin\qmake qscintilla.pro
6)	nmake (builds release version)
7)	nmake debug (builds debug version)
8)	copy debug qscintilla libs from Common\lib\ext\QScintilla\2.5.1\Qt4\debug to Common\lib\ext\QScintilla\2.5.1\lib\win32\debug
9)	copy release qscintilla libs from Common\lib\ext\QScintilla\2.5.1\Qt4\release to Common\lib\ext\QScintilla\2.5.1\lib\win32\release
10)	del Q:\common\lib\ext\qt\4.7.4\lib\*.lib (to remove the libs copied in step 1)


If rebuilding to change the name of the Qt or QScintilla libs, then also do:

1)	Update Q:\common\lib\ext\QT\Global-QT.props, and update the lib names
2)	Update Q:\common\lib\ext\QScintilla\Global-QScintilla.props, and update the lib name
3)	GDebugger -- update the following files with the new QT and QScintilla lib names:
	a)	InstallConfiguration.txt
	b)	InstallConfiguration-x64.txt
	c)	AMDTApplication.vcxproj (not QScintilla)
	d)	AMDTApplicationFramework.vcxproj (not QScintilla)
	e)	AMDTgDebuggerAppWrapper.vcxproj
	f)	GRApplicationComponents.vcxproj
	g)	gDebuggerAppCode.vcxproj
	h)	CopyFilesBeforeVSIXBuild.bat
	i)	CopyFilesToWorkEnvironment.bat
	j)	gDebuggerVSPackageVSIX-Debug.csproj
	k)	gDebuggerVSPackageVSIX-Release.csproj

Perforce (Qt):

1) Submit all changed files under the QT\4.7.4 directory
2) Mark for delete all existing files under QT\4.7.4\lib\win32
3) Mark for add all new files under QT\4.7.4\lib\win32
4) Mark for delete all project files that use the existing naming convention
4) Mark for add all project files that use the new naming convention
